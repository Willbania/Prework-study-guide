HTML

enter your HTML notes here

  • the header element is a container for metadata
  • the body element contains content that is shown to the user
    all of the elements that we want seen must be placed inside the body
  • the footer element goes at the bottom of the page
  • the title is a part of the head that is seen by the user
  • the p element represents a paragraph or block of text
  • list elements ol and ul represent ordered and un ordered lists
    each item of a list is seperated using the li element
  • the img element contains information about images that are displayed
    on the website
  • the img attribute is used for defining the location of media files sutch as
    images, videos and audio files
  • the br element creates a line of empty space or a line break between two blocks of content
  • the a tag creates links to the same webpage or other webpages.
  • the href attribute points to the url for the link in a similar way to the img src attribute
  • headings are created using the tags h1 through h6, each ascending number
    corresponds with a smaller heading size

GIT and Bash commands

bash commands

  • pwd - display current working directory
  • ls - list of files and folders in current directory
  • cd - navigates between directorys in the path. eg: cd prework-study-guide
  • cd .. - navigates backwards one stem through path
  • mkdir - creates new directory
  • touch - create new file
  • start - open file
  • code . - open VSCode

git commands

  • git status - check what branch you are currently on
  • git checkout -b branch name - creates a new branch and switches to it
  • git add -a - add all modifications in current working branch to the stagin area
  • git commit -m "Description of contents" - commits currently staged changes with -m to fla
  • git pull origin main - pull any changes from the origin of main branch
  • git push origin "current working branch" - push local changes to remote github branch
